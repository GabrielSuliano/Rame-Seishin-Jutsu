3. MESTRES.HTML (Tabela de Mestres)

Arquivo: paginas/mestres.html | CSS: css/mestres.css | JS: js/mestres.js

Funcionalidades:

  • ✅ Ordenação por coluna
  • ✅ Destaque de graus em vermelho

Endpoint:

GET /api/mestres
Resposta: [
  {
    id, grau, nome, posicao
  },
  ...
]

Graus válidos: HACHIDAN, ROKUDAN, GOKDAN, YONDAN, 
SANBAN, NIDAN, SHODAN, Dr., Srta.

4. CONTATO.HTML (Formulário)

Arquivo: paginas/contato.html | CSS: css/contato.css | JS: js/contato.js

Campos do Formulário:

Campo Tipo Validação
Nome text Min 3 chars, máx 100
Email email Formato válido
Telefone tel Caracteres válidos
Academia select Obrigatório
Mensagem textarea Min 10, máx 2000 chars

Endpoint:

POST /api/mensagens/enviar
Body: {
  nome: string,
  email: string,
  telefone: string,
  academia: string,
  mensagem: string
}
Response: { sucesso: boolean, mensagem: string }

5. LOGIN.HTML (Autenticação)

Arquivo: paginas/login.html | CSS: css/login.css | JS: js/login.js

Fluxo de Autenticação:

  1. Usuário acessa `/paginas/login.html`
  2. Sistema verifica localStorage
  3. Se autenticado, redireciona para homepage
  4. Caso contrário, mostra formulário
  5. Após login, salva usuário em localStorage

Endpoint:

POST /api/auth/login
Body: {
  usuario: string,
  senha: string
}
Response (sucesso): {
  sucesso: true,
  usuario: { id, nome, email, tipo },
  token?: string
}
Response (erro): {
  sucesso: false,
  erro: "Mensagem de erro"
}

🚀 Como Integrar com o Backend

Passo 1: Configurar URL da API

// Em: js/api.js, linha 1
CONFIG_API.urlBase = 'https://seu-backend.com/api'

Passo 2: Implementar Endpoints

  • GET /api/academias - Retorna lista de academias
  • GET /api/mestres - Retorna tabela de mestres
  • POST /api/mensagens/enviar - Recebe contato
  • POST /api/auth/login - Autentica usuário

Passo 3: Configurar CORS

Access-Control-Allow-Origin: https://seu-frontend.com
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Allow-Credentials: true

Passo 4: Testar Cada Página

  1. ✅ Homepage → Academias carregam?
  2. ✅ Mestres → Tabela mostra dados?
  3. ✅ Contato → Validação + envio funciona?
  4. ✅ Login → Autenticação persiste?

🔐 Segurança

⚠️ Validações Frontend: São apenas para UX. Sempre validar no backend também!

Recomendações:

  • ✅ HTTPS obrigatório em produção
  • ✅ CORS configurado para domínio específico
  • ✅ Senhas NUNCA em localStorage (usar httpOnly cookies)
  • ✅ Tokens JWT com expiração de 24h
  • ✅ Rate limiting em endpoints críticos
  • ✅ Validar ALL inputs no backend

🐛 Debugging

Console do Navegador (F12):

  • Ativa CONFIG_API.modoDebug = true para logs
  • Veja requisições na aba Network
  • Verifique localStorage na aba Application

Checklist:

  • [ ] URL da API está correta?
  • [ ] CORS habilitado?
  • [ ] Response JSON é válido?
  • [ ] Validações funcionam?
  • [ ] Login persiste?
  • [ ] Redireciona após ações?
